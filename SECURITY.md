# Security Policy

## Reporting a Vulnerability

Do **not** open a public issue for security bugs. A public report exposes the
flaw before a fix ships.

Email the security contact instead:

```text
nish@ubundi.co.za
```

Include in your report:

- What the issue is and the impact you expect.
- Steps to reproduce, or a proof of concept.
- Affected version, commit, or environment.
- Any suggested fix or mitigation.

## Acknowledgement

You will receive an acknowledgement within **3 business days**, followed by an
assessment and a remediation timeline.

## Supported Versions

Only the latest release on `main` receives security fixes.
